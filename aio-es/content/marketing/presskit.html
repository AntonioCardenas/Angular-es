NOMBRES DE MARCA

Angular

El nombre Angular representa el trabajo y las promesas proporcionadas por el equipo de Angular.

Cuando no se especifica, se supone que Angular se refiere a la última y mejor versión estable del Equipo Angular.

Ejemplo

Versión v4.1 ahora disponible - Nos complace anunciar que la última versión de Angular ya está disponible. ¡Mantenerse actualizado es fácil!

Ejemplo

Correcto: "Nuevas *ngIf capacidades—la nueva versión 4.0 tiene la capacidad de ...""

Incorrecto: "Nuevas *ngIf capacidades en Angular 4—Angular 4 tiene la capacidad de ..."

Razonamiento

Al no utilizar "Angular 4" en el título, el contenido sigue siendo aplicable y útil después de que se hayan lanzado las versiones 5, 6 y 7, ya que es poco probable que la sintaxis cambie a corto y mediano plazo.

AngularJS

AngularJS es la serie v1.x de trabajo y promesas proporcionadas por el equipo de Angular.

Ejemplos

  1. AngularJS es uno de los marcos de referencia(framework) más utilizados en la web hoy en día (por número de proyectos).
  2. Millones de desarrolladores están construyendo actualmente con AngularJS.
  3. Los desarrolladores están comenzando a actualizar de AngularJS a Angular.
  4. Estoy actualizando mi aplicación de AngularJS a Angular.
  5. Estoy usando material de AngularJS en este proyecto.

Los proyectos de AngularJS deberían usar el logo original de AngularJS / icono, y no el icono angular.

Angular Material

Este es el trabajo que realiza el equipo de Angular para proporcionar componentes de diseño de materiales para aplicaciones de angular .

Material AngularJS

Este es el trabajo que realiza el equipo de Angular en componentes de diseño de materiales que son compatibles con AngularJS.

Proyectos de terceros

X para Angular

Los terceros deben usar la terminología "X para Angular" o "ng-X" para proyectos de software. Los proyectos deben evitar el uso de Angular X (por ejemplo, Kit de herramientas de IU angular), ya que podría crear confusión de autoría. Esta regla no se aplica a eventos o grupos de reunión.

Los desarrolladores deben evitar el uso de números de versión Angular en los nombres de proyectos, ya que esto limitará artificialmente sus proyectos al vincularlos a un punto en el tiempo de Angular, o requerirá un cambio de nombre con el tiempo.

Cuando se usa un nombre en clave o un nombre corto, como en npm o github, algunos son aceptables, otros no son aceptables.

No se debe usar
  • ng2-
  • angular2-
Se puede usar
  • ng-
  • angular-

Como siempre, los selectores de componentes y directivas no deberían comenzar con selectores "ng-", ya que esto entrará en conflicto con los componentes y las directivas proporcionadas por el equipo angular.

Examples

  • El equipo ng-BE acaba de lanzar ng-health para ayudar a los desarrolladores a rastrear su propia salud.
  • Voy a usar NativeScript para Angular para aprovechar los widgets nativos de la Interfaz de Usuario (UI) .
  • HerramientaRealmenteGenial para Angular.
  • HerramientaRealmenteGenial para AngularJS.
